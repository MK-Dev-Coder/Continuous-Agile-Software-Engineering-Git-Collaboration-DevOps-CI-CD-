Pull Request Checklist

- [ ] Target branch is `develop` (feature branches -> develop -> main)
- [ ] Linked to an issue or requirement
- [ ] Includes tests or updates existing tests
- [ ] No secrets or credentials committed
- [ ] Commit messages follow project standards (clear, explain *why*)
- [ ] CI passes (build/tests)
- [ ] At least one reviewer has approved

Code Review Checklist

- [ ] Code is readable and follows style guidelines
- [ ] No large unrelated changes in a single commit
- [ ] Documentation updated where necessary
- [ ] Database migrations (if any) are included and documented
- [ ] Tests cover new behavior and pass

Use this file as a PR checklist or copy its content into .github/PULL_REQUEST_TEMPLATE.md after the .github file is backed up.
